## PART I — LIVE-SESSION PROTOCOL

### §1 — Session Opening Protocol

Every ad-hoc session begins with:

1. **Read MARSYS_JIS_BOOTSTRAP_HANDOFF.md** (once per session)
2. **Read PROJECT_ARCHITECTURE_v2_1.md relevant sections**
3. **Read SESSION_LOG.md** last 3 entries (establish current position)
4. **Clarify query type** (per Part II taxonomy below)
5. **Confirm scope** with native before deep analysis

### §2 — Query-Routing Protocol

Per Architecture §H.4 Whole-Chart-Read Protocol:

**Phase 1 — Whole-Chart (mandatory)**:
- UCN_v1_1 relevant Part(s)
- MSR high-confidence signals (MSR.413, MSR.391, MSR.396, MSR.404, MSR.397)
- CDLM relevant row

**Phase 2 — Focused**:
- Domain Report(s) for the specific domain(s)
- Matrix (L2) for forensic depth if needed

**Phase 3 — Synthesis**:
- Integrate whole + focused
- Flag tensions
- Cite sources
- Declare confidence

### §3 — Session Closing Protocol

Every ad-hoc session closes with:

1. **Confidence statement**: explicit 0.00-1.00 rating with reasoning
2. **Falsifier**: what would make this answer wrong?
3. **Gap flag**: which known gaps affect this answer?
4. **Follow-up registration**: what questions remain open?
5. **SESSION_LOG update**: brief entry in session log

## PART II — QUESTION TAXONOMY

Questions are classified into categories that determine routing:

### §4 — Category A: Structural Questions

"What does the chart say about X?" — structural characterization queries.

**Routing**: UCN → MSR → Domain Report (if narrow). High confidence possible.

**Examples**:
- "What is my primary career instrument?" → UCN §IV + MSR.413 → Answer: Mercury
- "How does my chart treat authority?" → UCN §VII (ATT pattern)
- "What is my soul's orientation?" → UCN §II (Soul-Trajectory)

### §5 — Category B: Decision-Support Questions

"Should I do X?" — specific choice guidance.

**Routing**: Full Whole-Chart-Read + relevant Domain Report(s). Moderate confidence; heavy caveat that chart reveals structure, not predetermined outcome.

**Examples**:
- "Should I take this job offer?" → REPORT_CAREER_DHARMA + current-dasha-window + Alignment-with-Mercury-deployment check
- "Should we have children now?" → REPORT_CHILDREN + REPORT_RELATIONSHIPS + current-timing (7H prerequisite status)

### §6 — Category C: Timing Questions

"When should I do X?" — muhurta-type queries.

**Routing**: UCN §VIII + HEATMAP_VARSHPHAL + Panchang muhurta check.

**Examples**:
- "When should I launch this product?" → 36-month heatmap + Wednesday-vara + Muhurta for the type of launch
- "When should I begin a major commitment?" → Saturn AD vs. post-Saturn AD phase implications

### §7 — Category D: Assessment Questions

"Is X healthy / aligned / working?" — state assessment queries.

**Routing**: Relevant Domain Report + MSR signals + current-window timing.

**Examples**:
- "Is this relationship in a good state?" → REPORT_RELATIONSHIPS + ATT assessment + current BB-UL window context
- "Am I managing health well?" → REPORT_HEALTH + Vata-leading-indicator check + current window

### §8 — Category E: Meta / Existential Questions

"What is my dharma / what is my purpose?" — highest-level questions.

**Routing**: UCN §X (Operating Instructions) → UCN §II (Soul-Trajectory) → UCN §X Chart's Central Request.

**Examples**:
- "What is the chart asking me to become?" → UCN §X: "Be specifically what you are built to be"
- "What is my ultimate life-purpose?" → UCN §III FS3 (Dharma-to-Moksha Trajectory) + Karakamsa-Mercury mission

### §9 — Category F: Clarification / Meta-Methodological Questions

"How does this system work?" / "Why do you interpret X as Y?"

**Routing**: Architecture document + relevant L2/L2.5 derivation ledger entries.

**Examples**:
- "Why is Mercury considered the chart's primary planet?" → MSR.413 seven-system derivation
- "What does Sade Sati actually mean for my chart?" → MSR.396 Paradox + UCN §VII.4

### §10 — Category G: External / Non-Chart Questions

Questions that are not chart-specific should be deflected or routed to non-MARSYS-JIS sources:
- "What is the best Sanskrit translation of X?" → external scholarly sources
- "What did my grandfather's chart look like?" → outside scope (different native)

## §11 — QUALITY STANDARDS

Every answer must:
- Begin with UCN-level framing (not domain-jump)
- Cite specific sources (UCN §, MSR.ID, CDLM.Dx.Dy, Report §)
- Declare confidence explicitly
- Acknowledge known gaps
- Provide falsifier
- Avoid prediction language ("X will happen") in favor of structural language ("the chart is configured for X")
