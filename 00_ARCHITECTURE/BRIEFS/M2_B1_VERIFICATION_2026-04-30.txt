§4 — Extractor implementation (AC.4)
==========================================================================

File: platform/python-sidecar/pipeline/extractors/cgm_extractor.py

Complete rewrite replacing manifest-loading with 15 derivation parser functions.

Key design decisions:
  - _make_edge() validates BOTH source AND target against merged node catalog
    (old code only checked target — root cause of orphan edges)
  - 15 independent parser functions, each returning list of edge dicts
  - _parse_sec_references: scans UCN content_excerpt for planet names in prose
    (UCN uses "Mercury", "Saturn" not "PLN.MERCURY") via regex, maps to PLN.* node IDs
  - _parse_resonates_with: planet-pair co-occurrence in 2+ YGA nodes (bidirectional)
    + hardcoded planet-house resonances from RM analysis
  - _parse_contradicts_with: 5 hardcoded pairs from CGM §5 structural paradoxes
    (self-loop removed: Mercury→Mercury skipped)
